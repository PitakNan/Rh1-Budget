ภาพรวมระบบ (System Overview)

สรุป Requirement ทั้งหมดจากไฟล์ แบ่งเป็น 7 โมดูลหลัก

1. Dashboard ผู้บริหาร (หน้าแรก)

แสดงภาพรวมงบประมาณแบบ Card/KPI พร้อม Drill-down ไปถึงระดับโครงการ มี API สำหรับ Embed ในเว็บอื่น

งบคงเหลือ ระหว่างดำเนินการ เสร็จสิ้น จองเงิน

2. จัดการแหล่งเงิน

CRUD แหล่งเงิน + โอนเงินระหว่างแหล่ง (ปีเดียวกัน) + ระงับ/ยกเลิกระงับ + Log ทุกการดำเนินการ (ต้องใส่รหัส PIN)

3. จัดการหน่วยบริการ

Master data รหัส 5 หลัก (Primary Key), ชื่อ, ระดับ รพ., จังหวัด — 215+ หน่วยบริการ ใน 8 จังหวัดภาคเหนือ

4. จัดการโครงการ

178 โครงการ (2566-2568) — เพิ่ม/แก้ไข/ยกเลิก/กลับคืน — ข้อมูล 13 ฟิลด์ + สีตามสถานะ + Log

5. สถานะโครงการ

Master data สถานะ เช่น ดำเนินการแล้วเสร็จ, อยู่ระหว่างดำเนินการ, ยกเลิก — CRUD + ระงับ

6. งบที่ใช้เกี่ยวข้องกับ

Master data ประเภทงบ 28 ประเภท เช่น นโยบายผู้บริหาร, โรคไม่ติดต่อเรื้อรัง, สุขภาพจิต — CRUD + ระงับ

7. ระบบสนับสนุน

Import/Export ข้อมูล (Backup), Web App Responsive รองรับ Smartphone, UI สไตล์ผู้บริหาร พร้อมสัญลักษณ์กระทรวงสาธารณสุข + สำนักงานเขตสุขภาพที่ 1

ข้อสังเกตสำคัญ: ทุกฟังก์ชันที่เป็น destructive (ยกเลิก, ระงับ, โอนเงิน, Import/Export) ต้องยืนยันด้วยรหัส PIN ก่อนดำเนินการ

LayerTechnology
FrontendReact / Next.js + Tailwind CSS
BackendNode.js (Express) หรือ Next.js API Routes
DatabasePostgreSQL
AuthPIN-based middleware (ไม่ต้อง Login ซับซ้อน)

API (Embed)REST API + CORS สำหรับ iframe/embed

LayerTechnology
FrontendAngular + PrimeNG
Backend.NET Core / Spring Boot
DatabaseMS SQL Server / PostgreSQL
AuthJWT + PIN verification layer

APIREST API + Swagger docs

2API สำหรับ Embedเว็บที่จะ Embed Dashboard คือเว็บอะไร? ต้อง CORS whitelist domain ไหน? ใช้ iframe หรือ API call?
3Data Migrationข้อมูล 178 โครงการจาก Excel จะ import เข้าระบบครั้งแรกอย่างไร? วันที่ใน Excel มีหลาย format ต้อง clean

5Multi-fund Source ต่อโครงการมีบางโครงการที่ใช้เงินจากหลายแหล่ง (เช่น โครงการ 56 = สนข.3.5 ลบ. + รพ.เกาะคา 1.5 ลบ.) ต้องรองรับไหม?

8Hostingต้อง deploy ใน server ของกระทรวง/เขตสุขภาพ หรือ Cloud ได้?
